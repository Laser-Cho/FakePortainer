# 프로젝트 목표
Portainer와 유사한 경량 웹 기반 Docker 컨테이너 관리 GUI 애플리케이션 개발

# 기술 스택
- **Backend:** Node.js (Express) + `dockerode` 라이브러리
- **Frontend:** Next.js (React) + Tailwind CSS + Shadcn UI
- **통신:** REST API (상태 조회 및 제어용), WebSocket (실시간 로그 스트리밍용)

# 핵심 기능 요구사항
1. **대시보드:** 호스트의 전체 컨테이너(실행/중지) 및 이미지 갯수 상태 요약
2. **컨테이너 관리:** 
   - 목록 조회 (ID, 이름, 매핑된 포트, 이미지명, 현재 상태)
   - 제어 액션 (Start, Stop, Restart, Remove)
3. **로그 뷰어:** 특정 컨테이너의 실시간 터미널 로그 출력
4. **이미지 관리:** 로컬 Docker 이미지 조회 및 사용하지 않는 이미지 삭제

# 아키텍처 및 제약 사항
- 백엔드는 호스트의 `/var/run/docker.sock`을 마운트하여 Docker 데몬과 직접 통신해야 함.
- 코드 구조는 백엔드(`api/` 폴더)와 프론트엔드(`web/` 폴더)로 분리할 것.
- API 응답 실패(예: 권한 부족, 컨테이너 없음 등) 시 프론트엔드에 명확한 에러 메시지를 전달할 것.
- UI는 깔끔한 다크 모드를 기본으로 구성할 것.
- **보안 사항:** 호스트 Docker 소켓의 높은 권한을 보호하기 위해 기본 패스워드 혹은 토큰 기반 인증 프로세스를 적용할 것.

# 단계별 작업 마일스톤 (Agy, 아래 순서대로 하나씩 확인을 받으며 진행해 줘)
- [Phase 1] 초기 셋업: 프론트엔드와 백엔드의 기본 폴더 구조를 생성하고, `package.json` 세팅 및 필요 라이브러리를 설치해 줘.
- [Phase 2] 백엔드 코어: Docker Socket과 연결하여 전체 컨테이너 목록을 JSON으로 반환하는 테스트 API를 만들어 줘.
- [Phase 3] 프론트엔드 UI: API를 호출하여 컨테이너 목록을 화면에 표(Table) 형태로 렌더링해 줘.
- [Phase 4] 제어 액션: UI에 Start/Stop 버튼을 추가하고, 이를 처리하는 API 라우트를 연동해 줘.
- [Phase 5] 실시간 로그: WebSocket 기반으로 특정 컨테이너의 터미널 로그를 실시간으로 띄워주는 모달 창을 구현해 줘.
- [Phase 6] 이미지 관리: 로컬 Docker 이미지를 조회하고, 사용하지 않는 이미지를 삭제할 수 있는 API 및 UI를 구현해 줘.
- [Phase 7] 보안 및 인증: 기본 패스워드 혹은 JWT 토큰 기반 인증 기능을 적용하여 비인가 접근을 차단해 줘.
